# TypeScript Migration — Commit Plan Design

**Date:** 2026-03-30 **Status:** Approved

## Context

The repository is mid-migration from a JavaScript implementation (`index.js`, `lib/parser.js`) to a full TypeScript rewrite (`src/`, `test/`, `tsconfig.json`, `tsup.config.ts`). The old JS files have been deleted and staged, but nothing has been committed. All new TS files are untracked. Several config files (package.json, eslint, workflows, gitignore) have been updated to support the new implementation.

A `tsx-501/` cache directory exists at the root — it is auto-generated by the `tsx` runtime and must be gitignored before any staging.

## Goal

Create a feature branch and land the entire migration as 7 atomic semantic commits, producing a clean, readable git history that clearly shows each concern of the migration independently.

## Branch

`feat/typescript-migration` — branched from `main`

## Commit Sequence

| # | Commit message | Files |
| --- | --- | --- |
| 1 | `chore: add tsx-501 to gitignore` | `.gitignore` |
| 2 | `chore: update github community files and workflow configs` | `.github/CODE_OF_CONDUCT.md`, `.github/ISSUE_TEMPLATE/bug_report.md`, `.github/ISSUE_TEMPLATE/feature_request.md`, `.github/workflows/pr-lint.yml`, `.github/workflows/stale.yml`, `.github/workflows/sync-labels.yml` |
| 3 | `ci: add ci and publish workflows` | `.github/workflows/ci.yml`, `.github/workflows/publish.yml` |
| 4 | `chore: migrate build tooling to typescript` | `package.json`, `package-lock.json`, `tsconfig.json`, `tsup.config.ts`, `eslint.config.mjs`, `.prettierrc.json` |
| 5 | `feat!: rewrite implementation in typescript` | `src/` (add), `index.js` (delete), `lib/parser.js` (delete) |
| 6 | `test: add test suite` | `test/` |
| 7 | `docs: add claude.md and project documentation` | `CLAUDE.md`, `docs/` |

## Constraints

- `tsx-501/` is gitignored before any `git add` — it must never appear in any commit
- `--no-verify` is never used
- No amending published commits
- Commits are staged explicitly by file/directory, not with `git add -A`
